# 竞品监控日报 Prompt

请基于输入 JSON 生成中文 Markdown 日报。只使用输入数据，不要编造。

硬性结构要求：输出必须严格包含以下五段，标题文字保持一致：

```markdown
# 竞品监控日报 · {北京时间日期}

## 一、总览

## 二、BSR 监测

## 三、站外每日发现

## 四、建议

## 五、注意
```

内容要求：
- 总览：2-4 句话，概括今天站外整体态势，并指出最值得注意的 1 个异动。
- BSR 监测：说明 BSR 口径为 `bsr_category.name`。先写 `bsr_monitor` 中已指定 ASIN 的排名；再用 `bestseller_monitor` 简要说明类目榜单的新进榜 ASIN、快速上升 ASIN。BSR 数值下降代表排名变好，数值上升代表排名变差。缺今天或昨天数据时明确写“数据缺失”。
- 站外每日发现：这是重点，按品牌组织，不按字段罗列。优先写 Diveblues，再写重点监控品牌，非重点品牌归入“其它品牌”。
- 站外每日发现至少回答：今天 Slickdeals / hip2save 上共有几家手持风扇品牌出现；促销价格区间；最低价品牌和价格；各重点品牌铺了几个 Deal、价位段、折扣力度。
- `offsite` 输入已经过滤为手持风扇品类；不要把暖手宝或其它品类写进日报。
- 促销价格区间只能使用 `offsite.price_range`，不要从标题年份、mAh、评论数等其它数字推断价格。
- 建议：给 1-3 条可执行站外运营建议，必须基于当天数据。
- 注意：列出数据缺失、抓取异常、需人工复核的点。
- 不要输出 Amazon 价格表、评分表、评论数表，也不要写“今日新增 ASIN”。
- 如果字段为空，写“暂无数据”或跳过，不要填 0。

